# Aurelia User Guide

This guide is written for site owners and teams. It explains the core workflow in plain language.

## 1. Create your workspace

Start with a clear business name and industry. A focused name helps the dashboard, search settings and public page feel consistent.

## 2. Build the first page

Open the builder and review the sections. Keep the headline direct, the first call-to-action visible and the product or service promise easy to understand.

## 3. Add products and visuals

Use the product manager for names, prices, stock and descriptions. Upload polished images in the asset library so your public page feels trustworthy.

## 4. Prepare search presence

Open Search Presence and review:

- title
- description
- keywords
- canonical URL
- open graph image
- structured data type

Good titles are specific. Good descriptions explain who the page is for and why it is useful.

## 5. Publish and measure

Publish from the builder. Visit the public URL, test the lead form and review analytics after traffic arrives.

## 6. Improve carefully

Use version history before large changes. Create an experiment when you want to test a different headline, offer or product section.

## Suggested launch checklist

- [ ] Site name is clear
- [ ] Homepage headline is direct
- [ ] Products have prices and stock
- [ ] Search title and description are complete
- [ ] Sitemap and robots files are available
- [ ] Lead form has been tested
- [ ] Custom domain is connected
- [ ] Team members have correct roles
